# lastCumulativeRate is not updated before calculating borrowerDebt

### Summary

[BorrowerDebt](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/BorrowLib.sol#L825) is calculated based on `lastCumulativeRate.`
However, `CumulativeRate` is [not updated](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/Core_logic/borrowing.sol#L657) before borrower` withdraw`, which leads to incorrect calculation of `borrowerDebt.`

### Root Cause

`BorrowerDebt` must be calculated before [withdraw](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/BorrowLib.sol#L825)  and [liquidation](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/Core_logic/borrowLiquidation.sol#L205).
As the following  liquidate flow，call calculate cumulative rate fucntion to get interest before calculating borrowerDebt
```solidity
    function liquidate(
        address user,
        uint64 index,
        IBorrowing.LiquidationType liquidationType
    ) external payable whenNotPaused(IMultiSign.Functions(2)) onlyAdmin {
        // Check whether the user address is non zero address
        if (user == address(0)) revert Borrow_MustBeNonZeroAddress(user);
        // Check whether the user address is admin address
        if (msg.sender == user) revert Borrow_CantLiquidateOwnAssets();

        // Call calculate cumulative rate fucntion to get interest
        calculateCumulativeRate();

```

 BorrowerDebt is calculated based on `lastCumulativeRate`.
However, `cumulativeRate` is not [updated](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/Core_logic/borrowing.sol#L374) before `borrower withdraw`, which leads to incorrect calculation of `borrowerDebt`.


Consider the following scenario：

1.user deposits 1 ETH as  collateral 

skip timeStamp + 2000

2.user withdraw Collateral from the protocol 
user normalizedAmount =799999998
`lastCumulativeRate`=1000000287765469344427564015
`borrowerDebt` = 1000000287765469344427564015 * 799999998) / 1e27 = 800000228.2123748

But the correct calculation borrowerDebt  is call [calculateCumulativeRate()](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/BorrowLib.sol#L229) fucntion to get interest.


After updating` lastCumulativeRate `correctly:
`lastCumulativeRate` = 1000003321684945746011616154
`borrowerDebt` = 1000003321684945746011616154 * 799999998 / 1e27 =800002655.3479499

    async function calculateUpsideToSwap(
        found:any,
        ethPrice: number,
        exchangeRate: number,
        inputTokenAddress: AddressLike,
        treasuryAddress: AddressLike,
        borrowAddress: AddressLike
    ) :Promise<{ odosData: any; odosSignData: any }>{
        
        if (ethPrice > Number(found.ethPriceAtDeposit)) {
            let priceDiff:any;
            const strikePrice = Number(found.strikePrice/found.depositedAmountInETH);
            if(ethPrice > strikePrice) {
                priceDiff = strikePrice - Number(found.ethPriceAtDeposit);
            }else {
                priceDiff = ethPrice - Number(found.ethPriceAtDeposit);
            }
            let upside = priceDiff * Number(found.depositedAmountInETH);
            upside = (upside / ethPrice) * 1e18;
            let upsideToSwap = (upside / exchangeRate).toString();
            upsideToSwap.slice(0, -1) + "0";
            const odosData = await fetchDataFromOdos(inputTokenAddress, treasuryAddress, upsideToSwap);
            const odosSignData = await signData(odosData, borrowAddress)
            return {odosData, odosSignData};
        } else {
            return { odosData: "0x", odosSignData: "0x"}
        }
      }
