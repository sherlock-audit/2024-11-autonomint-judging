# CDS User cannot withdraw Token due to incorrect update of totalCdsDepositedAmount

### Summary

CDS User cannot withdraw Token due to incorrect update of [totalCdsDepositedAmount](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/CDSLib.sol#L728) .

Incorrect update of `totalCdsDepositedAmount` lead to Arithmetic operation overflow , which makes users unable `withdraw `

### Root Cause


Let's say:
current ETH price = 100000

1.  `user1` deposit 10000000000 USDT to `CDSContractA` and 6000000000 USDT to `CDSContractB` (user1 opted for liquidation)

2.  `user2` deposit 10 ETH to `BorrowingContractA` and `BorrowingContractB`
current  chain A `totalAvailableLiquidationAmount` = 10000000000
current  chain  B`totalAvailableLiquidationAmount` = 6000000000

3.  ETH price down = 60000,liquidate user2 position
First, liquidate the `BorrowingContractB` position and [get liquidation amount](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/Core_logic/borrowLiquidation.sol#L281) 5125002359 from A chain .

  After liquidate `BorrowingContractB` position,
   chain A `totalCdsDepositedAmount ` = 4874997641 =  10000000000 - 5125002359([liqAmountToRemove](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/Core_logic/GlobalVariables.sol#L632) )
   chian A `totalAvailableLiquidationAmount` = 4874997641

4.  `user1`  withdraw token from A chain and [update totalCdsDepositedAmount](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/CDSLib.sol#L728)

`totalCdsDepositedAmount -= (params.cdsDepositDetails.depositedAmount - params.cdsDepositDetails.liquidationAmount)`

current ` user1` `cdsDepositDetails.depositedAmount` =10000000000
current  `user1` `cdsDepositDetails.liquidationAmount`= 4874997641
  chain A `totalCdsDepositedAmount ` = 4874997641


according to the above codeï¼š[update totalCdsDepositedAmount](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/CDSLib.sol#L728)
10000000000 - 4874997641 = 5125002359 > 4874997641,lead to Arithmetic operation overflow.



### Internal pre-conditions

_No response_

### External pre-conditions

1.User[ deposit](https://github.com/sherlock-audit/2024-11-autonomint/blob/0d324e04d4c0ca306e1ae4d4c65f0cb9d681751b/Blockchain/Blockchian/contracts/lib/BorrowLib.sol#L644) collateral into the protocol
2.When ETH price down ,liquidate the the borrower's position and get liquidation amount from other chain
3.CDS  user withdraw the deposited amount

### Attack Path

1.User deposit collateral into the protocol on multiple chain
2.When ETH price down ,liquidate the the borrower's position(chain B) and get liquidation amount from other chain(chain A)
3.CDS  user withdraw the deposited amount from  chain A ,which can lead to withdraw revert

### Impact

This will DoS some users from withdrawing their tokens due to arithmetic overflow

        found:any,

        exchangeRate: number,
        inputTokenAddress: AddressLike,
        treasuryAddress: AddressLike,
        borrowAddress: AddressLike
    ) :Promise<{ odosData: any; odosSignData: any }>{
        
        if (ethPrice > Number(found.ethPriceAtDeposit)) {
            let priceDiff:any;
            const strikePrice = Number(found.strikePrice/found.depositedAmountInETH);
            if(ethPrice > strikePrice) {
                priceDiff = strikePrice - Number(found.ethPriceAtDeposit);
            }else {
                priceDiff = ethPrice - Number(found.ethPriceAtDeposit);
            }
            let upside = priceDiff * Number(found.depositedAmountInETH);
            upside = (upside / ethPrice) * 1e18;
            let upsideToSwap = (upside / exchangeRate).toString();
            upsideToSwap.slice(0, -1) + "0";
            const odosData = await fetchDataFromOdos(inputTokenAddress, treasuryAddress, upsideToSwap);
            const odosSignData = await signData(odosData, borrowAddress)
            return {odosData, odosSignData};
        } else {
            return { odosData: "0x", odosSignData: "0x"}
        }
